# Shipping Checklist

You've built something. Now the real question: what would it actually take to turn this prototype into something your organisation could use?

Work through this checklist as a group. You don't need answers to everything; the point is to show you've thought about it. This feeds directly into the "what it would take to ship" slide in your presentation.

---

## Security & Access

- [ ] Does the app handle any confidential or personally identifiable information?
- [ ] Who should be allowed to access it? (Everyone in the organisation? Just one team? A small group only?)
- [ ] Does it need to sit behind your organisation's login and authentication system?
- [ ] Where would it be hosted? (Internal infrastructure, approved cloud provider, internal tool platform?)

---

## Data Sources

- [ ] What data does the app rely on to work?
- [ ] Is that data currently available in a structured, accessible format?
- [ ] Would we need a data feed, an API, or a manual upload process?
- [ ] How would the data stay up to date - and who would be responsible for that?

---

## Integration

- [ ] Does this need to connect to any existing internal systems? (Document management, scheduling, email, finance)
- [ ] Are there any API access or licensing considerations?
- [ ] Would it replace an existing tool, or sit alongside it?

---

## Maintenance & Ownership

- [ ] Who would own this tool going forward?
- [ ] Who would fix it if it breaks or needs updating?
- [ ] Is there a budget or procurement consideration?
- [ ] Does it need to go through IT review or information security sign-off?

---

## The "So What"

- [ ] How much time would this save per user, per week - if it actually worked at scale?
- [ ] What would it take to get from this prototype to a pilot with a real team?
- [ ] Is there a team lead, sponsor, or decision-maker who would champion this?
- [ ] What's the minimum next step - who would you need to talk to on Monday?
